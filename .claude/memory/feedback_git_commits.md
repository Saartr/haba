Не делать `git commit` и `git push` автоматически после каждого изменения кода.

**Why:** Пользователь хочет контролировать когда и что коммитится. Автоматические коммиты на каждое изменение — лишний шум в истории и раздражает.

**How to apply:** Коммитить и пушить только когда пользователь явно просит ("закоммить", "запушь", "сохрани в git"). При деплое через `deploy-backend.ps1` тоже не пушить автоматически — деплой работает через SSH pull, не через push.

Сообщения коммитов писать **по-русски**: заголовок и тело. Префиксы (`feat:`, `fix:`, `refactor:` и т.д.) оставлять на латинице — это конвенция. Пример:
```
feat: передача прав создателя группы

- habit/[id].tsx: шторка выбора участника, подтверждение, handleTransfer
- BottomSheet.tsx: children рендерятся только при visible=true
```
